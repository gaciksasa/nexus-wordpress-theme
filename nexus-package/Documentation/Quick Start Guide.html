Requirements

Before installing Nexus, make sure your server meets the following requirements:

WordPress6.3 or higher
PHP8.0 or higher
Memory Limit256 MB or higher
Max Upload Size16 MB or higher (for theme ZIP upload)
Max Execution Time180 seconds (for demo import)

Recommended Plugins

  • Elementor (free or Pro) — page builder for visual editing
  • Kirki Customizer Framework — powers the theme Customizer options
  • WooCommerce — required only if building an online store
  • CMB2 — custom meta boxes for pages and custom post types
All recommended plugins will be prompted for installation automatically via the setup wizard after theme activation.

Theme Installation

If you have downloaded and unzipped the full theme package, locate the theme installation file called nexus.zip.

Important: Upload nexus.zip, NOT the full package ZIP file. The full package contains documentation, licensing, and other files that WordPress cannot process as a theme.

Upload via WordPress Admin

  1. Log in to your WordPress admin area.
  2. Navigate to Appearance > Themes.
  3. Click Add New, then click Upload Theme.
  4. Choose the nexus.zip file from your computer and click Install Now.
  5. After installation completes, click Activate.
If the upload fails, your server may have a low upload file size limit. Contact your hosting provider to increase it to at least 16 MB, or use the FTP method below.

Upload via cPanel

  1. In cPanel File Manager, navigate to public_html/wp-content/themes/.
  2. Click Upload and upload the nexus.zip file.
  3. Once uploaded, select the ZIP file and click Extract.
  4. Go to Appearance > Themes in WordPress and activate Nexus.

Upload via FTP

  1. Unzip nexus.zip on your computer. You will get a folder called nexus.
  2. Using an FTP client (e.g., FileZilla), connect to your server.
  3. Upload the nexus folder to wp-content/themes/.
  4. Go to Appearance > Themes in WordPress and activate Nexus.

Setup Wizard

After activating Nexus, you will be redirected to the setup wizard. If you missed it, navigate to Appearance > Nexus Setup.

The wizard guides you through the following steps:

  1. Welcome — Introduction and getting started.
  2. Plugins — Install and activate recommended plugins (Elementor, Kirki, CMB2, WooCommerce).
  3. Demo Import — Choose and import one of 15 pre-built demo websites.
  4. Ready — Your site is set up and ready to customize!
We strongly recommend running the setup wizard to get the best experience with Nexus. You can always skip steps and come back later.

WooCommerce Integration

Nexus provides full WooCommerce support with custom styling and enhanced features:

  • HPOS Compatible — Supports WooCommerce High-Performance Order Storage
  • Off-Canvas Cart — Slide-out mini cart accessible from the header
  • Grid/List View Toggle — Customers can switch between grid and list views on shop pages
  • Product Carousel Widget — Elementor widget to display products in a Swiper.js carousel
  • Custom Templates — Styled archive, single product, mini-cart, and checkout templates

WooCommerce settings are available in Appearance > Customize > WooCommerce.

Dark Mode

Nexus includes a built-in dark mode system:

  • Toggle via the Customizer: Appearance > Customize > Dark Mode
  • Automatic detection using prefers-color-scheme media query
  • User preference saved in localStorage for persistence
  • Applied via the .nexus-dark CSS class on the <html> element

All Nexus widgets and theme components are fully styled for both light and dark modes.

Child Theme

A child theme allows you to make customizations that persist through theme updates. We strongly recommend using the included child theme for any custom CSS or PHP modifications.

Installation

  1. Locate nexus-child.zip in the Developer Tools folder of the full package.
  2. Upload and install it via Appearance > Themes > Add New > Upload Theme.
  3. Activate the Nexus Child theme.

Customization

  • Custom CSS — Add to nexus-child/style.css
  • Custom PHP — Add to nexus-child/functions.php
  • Template Overrides — Copy any template file from nexus/ to nexus-child/ and modify it

Troubleshooting

Cannot upload the theme

If the upload fails, your server's upload file size limit is too low. Contact your hosting provider and ask them to increase it to at least 16 MB. Alternatively, install the theme via FTP or cPanel.

Missing the style.css stylesheet

This error means you uploaded the wrong file. Make sure you upload nexus.zip (the installable theme file), NOT the full package ZIP. The installable file is located in the root of the package folder.

Blank page after activation

This is usually caused by a PHP memory limit issue. Add the following to your wp-config.php file:

define( 'WP_MEMORY_LIMIT', '256M' );

If this does not help, contact your hosting provider to increase the PHP memory limit.

404 errors on pages

After theme activation or demo import, navigate to Settings > Permalinks and click Save Changes (without changing anything). This refreshes the rewrite rules.

Elementor widgets not showing

Make sure the Elementor plugin is installed and activated. Nexus widgets appear under the Nexus category in the Elementor editor panel. If widgets are missing, deactivate and reactivate the Nexus theme.

FAQ

Do I need Elementor?

No. Nexus works with both Elementor and the default WordPress Gutenberg editor. However, Elementor is recommended for the best experience with all 22 custom widgets.

Is WooCommerce required?

No. WooCommerce is only needed if you want to run an online store. The theme works perfectly without it.

How do I update the theme?

Download the latest version from ThemeForest, then upload nexus.zip via Appearance > Themes > Add New > Upload Theme. WordPress will prompt you to replace the existing version.

Is the theme RTL ready?

Yes. Nexus includes a dedicated rtl.css stylesheet that is automatically loaded for RTL languages.

Is the theme translation ready?

Yes. Nexus includes a .pot file in the languages/ folder for translation using tools like Poedit, Loco Translate, or WPML.

Where can I get support?

You can get support through the item's comments section on ThemeForest or by contacting us through our support page.
